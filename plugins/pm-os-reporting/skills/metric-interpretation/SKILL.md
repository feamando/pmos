# Metric Interpretation

## When to Apply
- Presenting any metric with a change (WoW, YoY, MoM)
- Analyzing performance data in reports or updates
- Responding to questions about metric movements

## Core Principle
**Explain WHY metrics moved, not just the numbers.** A metric without context is noise. A metric with a hypothesis is signal.

## Interpretation Framework

### 1. State the Fact
- What metric, what value, what change
- Example: "Orders: 7,476 (-1% WoW; +172% YoY)"

### 2. Hypothesize the Cause
- Connect to known events, launches, experiments, or external factors
- Example: "Decline linked to reverting to BAU offer after promotional period"

### 3. Assess Significance
- Is this within normal variance or a meaningful shift?
- Compare against baseline, not just previous period
- Flag if the trend has persisted for 2+ periods

### 4. Recommend Action
- If declining: what to investigate or change
- If growing: what to sustain or accelerate
- If stable: whether that's good (on target) or bad (stagnant)

## Common Metric Patterns

| Pattern | Likely Cause | Action |
|---------|-------------|--------|
| Sharp spike then drop | Promotional period ended | Compare against pre-promo baseline |
| Gradual decline over 3+ weeks | Structural issue or market shift | Deep dive by channel/segment |
| WoW flat but YoY up strongly | Healthy growth, normal fluctuation | No action needed |
| Divergent channels | Channel-specific issue | Isolate and investigate the outlier |
| Metric up but KPI down | Vanity metric or funnel leak | Check downstream conversion |

## Anti-Patterns
- Reporting numbers without comparison periods
- Celebrating a WoW increase that's within noise
- Ignoring YoY context when WoW looks bad
- Attributing changes to a single cause without evidence
- Using averages when medians would be more representative
