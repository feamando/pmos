# 1:1 Meeting Structure

## When to Apply

- User prepares for a 1:1 meeting
- User asks for 1:1 prep or agenda
- User summarizes a 1:1 meeting
- User discusses 1:1 cadence or format

## Standard Structure

Every 1:1 follows this flow:

1. **Check-in** — "What's top of mind?" Open-ended, let them lead
2. **Blockers** — "What's stopping progress?" Identify and commit to removing obstacles
3. **Projects** — Quick status on key initiatives (not a status meeting — focus on decisions needed)
4. **Development** — Coaching, feedback, career growth (not every meeting, but regularly)
5. **Asks** — "What do you need from me?" and state what you need from them

## Prep Guidelines

| Source | What to Extract |
|--------|----------------|
| Previous 1:1 notes | Open action items, unresolved topics |
| Daily context | Relevant blockers or updates for this person |
| Career plan | Development topics to check in on |
| Brain entity (if available) | Recent project associations, relationship context |

## Time Allocation

- **30-minute 1:1:** Check-in (3 min) + Blockers (5 min) + Projects (12 min) + Development (5 min) + Asks (5 min)
- **For reports:** Lean toward Development and Blockers
- **For manager:** Lean toward Projects and Asks
- **For stakeholders:** Lean toward Projects and alignment

## Output Format

```markdown
## 1:1 Prep — [Person Name] | [Date]

### Check-in
- [Carry-forward items from last meeting]
- [Personal context if known]

### Blockers
- [Known blockers relevant to this person]

### Projects
- **[Project]:** [Status / decision needed]

### Development
- [Career plan items to discuss]
- [Recent feedback to share]

### Asks
- [What I need from them]
- [What they might need from me]
```

## Anti-Patterns

- Using 1:1s as status meetings (use standups for that)
- Skipping the check-in to "get to business"
- Not following up on action items from previous meetings
- Only discussing projects, never development
